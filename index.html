Executive Summary

This Statement of Work outlines two implementation approaches for Kaorium's white-label CRM solution designed specifically for the fragrance industry. As "The Scent Sommelier" expanding from 400+ successful deployments in Japan to the UK and European markets, Kaorium requires a sophisticated CRM system that can manage B2B relationships with fragrance retailers while capturing and analysing consumer scent preference data.

Our Recommendation

Based on Kaorium's specific requirements for fragrance industry integration, white-label capabilities, and long-term scalability, we recommend the Custom Supabase/Next.js solution. While requiring higher initial investment, it provides superior flexibility for fragrance-specific features, better control over multi-tenant architecture, and significantly lower long-term operational costs at scale.

White-Label & Multi-Tenant Architecture

Authentication & Access Control
  • Email & Password Authentication: Each retail partner's users log in with email/password via Supabase Auth
  • Tenant Isolation: Row-Level Security (RLS) ensures complete data separation between partners
  • Custom Login Pages: Each partner gets their own branded login URL (e.g., partner1.kaoriumcrm.com)
  • Role-Based Access: Admin, Manager, and User roles with granular permissions per tenant
White-Label Capabilities
  • Custom Domains: Each partner can use their own domain (e.g., crm.partner.com)
  • Dynamic Theming: Logo, colours, fonts stored per tenant and applied at runtime
  • Email Branding: Automated emails use partner's branding and from address
  • Dedicated Dashboards: Each partner sees only their data and custom KPIs
Selling to Other Companies
  • SaaS Model: Monthly subscription per retail partner based on contacts/features
  • Self-Service Onboarding: New partners can sign up and configure their instance
  • Automated Provisioning: New tenant setup in minutes with default configuration
  • Revenue Sharing: Optional model where partners pay based on campaign performance
